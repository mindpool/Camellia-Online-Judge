## 提交pull request

1. [Fork](https://github.com/mindpool/evergreen-online-judge/fork) 本仓库到你自己的GitHub个人账户中。
2. [Clone](https://docs.github.com/en/repositories/creating-and-managing-repositories/cloning-a-repository) 你 fork 的仓库到本地。
3. 从该项目贡献文档中找到`对应项目的主分支`拉出一个新分支。
4. 搭建开发环境，你可以阅读对应项目的文档来了解如何搭建开发环境。
5. 如果您修复了错误或添加了需要被测试的代码，则需要添加一些测试。
6. 请确保所有测试都通过。
7. 提交Pull Request， 确保所有的CI都通过。
8. 等待项目维护者的Review，如果有必要，您可能需要对您的代码进行一些修改，直到项目维护者确认您的代码可以合并。

提交 Pull Request 时，请注意以下事项：

- 保持 PR 足够小，以便每个 PR 只解决一个问题或添加一个功能。
- 请在 PR 中包含适当的描述，并链接相关问题。
